DOM Inspector — Guided Tutorial

The <dom-inspector> demonstrates event capturing vs bubbling and safe element highlighting.

Annotated snippet

// handler runs in capturing mode on document (true argument)
document.addEventListener('click', handler, true);
function handler(e){
  // 1. prevent default so page actions don't navigate away
  // 2. stopPropagation to keep the inspector from interfering with app handlers
  // 3. highlight and read information from e.target
}

Walkthrough

  1. Capture phase — registering with true ensures the inspector sees the event before other listeners, useful for debugging and safe highlighting.
  2. Prevent/stop — call e.preventDefault() and e.stopPropagation() so inspecting doesn't trigger application behaviors.
  3. Highlight — add a transient CSS class to the selected element for visual feedback and remove it after a short timeout.
