Widgets — Guided Tutorial

This tutorial explains how the <demo-tabs> and <drag-list> components are implemented and how they interact with the DOM.

Tabs (concepts)

  1. Create a shadow root to encapsulate styles and markup.
  2. Render the button list and panels inside the shadow DOM.
  3. Listen for clicks and toggle the hidden attribute on panels.
  4. Prefer event delegation inside the shadow root for simple APIs.

Walkthrough (line-by-line)

  1. Shadow root — calling this.attachShadow({mode:'open'}) prevents styles leaking and allows the component to define its own CSS.
  2. Render — the component writes static HTML for controls and panels into shadowRoot.innerHTML. Keep this minimal to improve performance.
  3. Event delegation — instead of wiring each button, add a single click handler on the shadow root and use e.target.matches('[data-id]') to detect the button pressed.
  4. Show method — calling show(id) hides all panels and un-hides the matching panel by toggling the hidden attribute. This keeps DOM updates simple and accessible.
  5. Accessibility — consider adding ARIA attributes (role="tablist", role="tab", aria-selected) when expanding this pattern.

Implementation tips

  • Keep data-bindings simple: components should reflect DOM state from attributes or properties.
  • Avoid heavy computation in connectedCallback; defer work to microtasks if needed.

Drag & Drop (notes)

The <drag-list> uses native dragstart, dragover, and drop events — on drop it reorders child nodes by moving the dragged node before/after the drop target.
