Walkthrough

  1. Render — building markup inside a shadow root isolates styles. Use minimal markup so the component can be reused.
  2. Persist — use a versioned key (e.g., demo-todos-v1) so future upgrades can migrate safely.
  3. Re-render — always re-render from the source-of-truth (localStorage) after mutations to avoid UI drift.
  4. Remove — remove handlers should update storage then re-render rather than mutating DOM only.
